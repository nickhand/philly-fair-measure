# Fair Measure, public dashboard

The Vue 3 + TypeScript front end for the Fair Measure assessment check, live
at [nickhand.dev/fair-measure](https://nickhand.dev/fair-measure/). It renders
the citywide map (MapLibre), the address search, and the per-property report
against the FastAPI backend in `src/philly_fair_measure/api.py`. Architecture
and deployment notes are in [docs/frontend.md](../docs/frontend.md).

## Development

```sh
npm install
npm run dev          # dev server on :5173, proxies /api to :8000
```

Start the API alongside it from the repository root:

```sh
uv run fair-measure api    # JSON API on :8000
```

## Checks

```sh
npm run test:unit    # Vitest
npm run type-check   # vue-tsc
npm run lint         # ESLint
npm run build        # type-check + production build
```

## Configuration

| Variable | Purpose | Default |
|---|---|---|
| `VITE_API_BASE` | Absolute base URL for API requests | empty (same-origin `/api`, proxied in dev) |
| `VITE_PUBLIC_BASE` | Base path the site is served under | `/` (production uses `/fair-measure/`) |

Production values live in [netlify.toml](../netlify.toml). Site-wide links and
the displayed assessment year are in `src/config/site.ts`. Displayed statistics
come from `src/data/siteStats.json`, which is committed and regenerated by
`uv run fair-measure export-web-stats` after each model run.
